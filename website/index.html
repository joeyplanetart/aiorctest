功能特性

一站式 AI 增强 API 测试

从接口管理到场景编排，从 AI 生成到结果断言，覆盖完整测试生命周期。

DAG 可视化编排

拖拽节点、连线组合多个 API 调用，系统自动按拓扑顺序执行，支持变量在节点间自动传递。

  • XYFlow 流程图画布
  • 节点间 JSONPath 数据映射
  • 拓扑排序自动执行

AI 场景智能生成

用自然语言描述测试需求，AI 从项目 API 库中自动选择接口、配置参数、设置断言，生成完整测试流。

  • 自然语言 → DAG 场景
  • 自动选择相关接口
  • 支持微调后保存

完整 API 管理

层级文件夹组织接口，支持 cURL 和 OpenAPI 批量导入，内置调试面板与响应断言配置。

  • cURL / OpenAPI 导入
  • JSONPath 断言规则

上传接口文档后，AI 可检索文档上下文辅助生成测试场景，让 AI 更理解你的业务逻辑。

工作原理

三步完成 AI 编排测试

从接口导入到场景运行，最快 5 分钟搭建完整测试链路。

01

导入你的 API

粘贴 cURL 命令或 OpenAPI JSON，批量导入项目接口。支持层级文件夹管理，配置环境变量替换域名。

bash 
# 支持直接粘贴 cURL
curl -X POST 'https://api.example.com/login' \
  -H 'Content-Type: application/json' \
  -d '{"email":"test@example.com"}'

# 或上传 OpenAPI / Swagger JSON
02

AI 生成或手动编排场景

用自然语言描述测试场景，AI 自动生成 DAG 节点与连线。也可在画布上拖拽手动搭建，配置变量提取与断言。

快速开始

5 分钟本地跑起来

无需复杂配置，一键启动脚本自动处理依赖安装与端口管理。

1
克隆并配置环境变量
bash 
git clone <repo-url> AIOrcTest && cd AIOrcTest
cp .env.example .env
# 编辑 .env，填写 OPENAI_API_KEY 和 SECRET_KEY
2
启动后端（自动安装依赖）
bash 
./scripts/start-backend.sh
# 后端就绪后访问 http://localhost:8000/docs
3
启动前端（新终端）
bash 
./scripts/start-frontend.sh
# 访问 http://localhost:5173
1
配置环境变量
bash 
cp .env.example .env
# 编辑 .env 填写 OPENAI_API_KEY 和 SECRET_KEY
2
创建 Dockerfile 并启动
bash 
docker compose up --build -d
# 后端运行在 http://localhost:8000

关键环境变量

变量必填说明
OPENAI_API_KEY 必填 OpenAI 或兼容 API 密钥
SECRET_KEY 必填 JWT 签名密钥（随机 32 字节串）
OPENAI_API_BASE 可选 自定义 API 基础 URL（OpenRouter / Azure 等）
DATABASE_URL 可选 默认 SQLite，生产可切换 PostgreSQL
BOOTSTRAP_SUPERADMIN_EMAILS 可选 启动时自动设为超级管理员的邮箱

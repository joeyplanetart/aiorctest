## 账号与登录

### 注册账号

1. 访问系统地址（默认 `http://localhost:5173`）
2. 点击「注册」，填写邮箱、密码和姓名
3. 注册成功后自动跳转至登录页

> **超级管理员说明：** 系统管理员可通过配置 `BOOTSTRAP_SUPERADMIN_EMAILS` 环境变量，在服务启动时自动将指定邮箱提升为超级管理员。

### 登录

输入邮箱和密码后登录。登录状态通过 JWT Token 维持，有效期默认 24 小时（可通过 `ACCESS_TOKEN_EXPIRE_MINUTES` 调整）。

### 语言切换

点击顶栏右侧地球图标，可在 **中文** 和 **English** 之间切换。

## 项目管理

项目是平台最顶层的隔离单元，所有 API、场景、环境变量都归属于特定项目。

### 创建项目

> 仅超级管理员可创建项目。

1. 进入「项目」页面
2. 点击「新建项目」
3. 填写项目名称和描述
4. 确认后，系统自动创建三个环境：`stage`（测试）、`pre`（预发）、`prod`（生产）

### 管理项目成员

在项目详情页 → 「成员」标签：

| 角色 | 权限 |
|------|------|
| `owner` | 完全控制，可管理成员 |
| `editor` | 可读写 API、场景、变量 |
| `viewer` | 只读，可运行场景 |

1. 点击「添加成员」，输入用户邮箱
2. 选择角色后确认
3. 可随时修改角色或移除成员

### 切换项目

左侧导航栏顶部显示当前项目，点击可切换到其他已加入的项目。系统会记住上次选择。

## API 管理

进入项目后点击左侧「API 管理」进入接口管理页面。

### 组织结构

API 通过**文件夹**组织，支持无限层级嵌套：

```
项目
└── 文件夹（如：用户模块）
    ├── 子文件夹（如：认证接口）
    │   ├── POST /auth/login
    │   └── POST /auth/register
    └── GET /users/{id}
```

**操作：**
- 右键文件夹 → 新建子文件夹 / 重命名 / 删除
- 文件夹内点击「+ 新建端点」

### 新建 API 端点

1. 选中目标文件夹，点击「新建端点」
2. 填写：
   - **名称**：描述性名称，如 `获取用户信息`
   - **方法**：GET / POST / PUT / PATCH / DELETE
   - **URL**：支持环境变量，如 `{{base_url}}/users/{{user_id}}`
   - **请求头**：Key-Value 对，如 `Authorization: Bearer {{token}}`
   - **请求体**：JSON 格式（POST/PUT/PATCH 时）
3. 保存

### 导入 cURL

适合从浏览器开发者工具直接复制 cURL 命令导入。

1. 点击「导入 → 从 cURL 导入」
2. 粘贴 cURL 命令，例如：

```bash
curl -X POST 'https://api.example.com/login' \
  -H 'Content-Type: application/json' \
  -d '{"email":"test@example.com","password":"123456"}'
```

3. 选择目标文件夹
4. 确认导入，系统自动解析方法、URL、Headers 和 Body

### 导入 OpenAPI

适合批量导入已有 API 文档。

1. 点击「导入 → 从 OpenAPI 导入」
2. 粘贴 OpenAPI 3.0 / Swagger 2.0 JSON
3. 系统自动按 `tags` 创建文件夹，批量导入所有端点
4. 确认导入

### 调试 API

1. 点击任意端点，右侧展开调试面板
2. 选择**运行环境**（stage / pre / prod）
3. 点击「发送」按钮
4. 查看响应：
   - 状态码（带颜色标识：2xx 绿色，4xx/5xx 红色）
   - 响应时间（毫秒）
   - 响应 Headers
   - 响应 Body（JSON 格式化显示）

### 配置断言

断言用于验证接口响应是否符合预期。

1. 在端点编辑页切换到「断言」标签
2. 点击「添加断言」
3. 配置断言规则：

| 字段 | 说明 | 示例 |
|------|------|------|
| 提取路径 | JSONPath 表达式 | `$.data.user.id` |
| 断言类型 | 比较方式 | `equals` / `contains` / `exists` / `regex` |
| 期望值 | 预期结果 | `123` |

4. 调试时断言结果会显示通过（✓）或失败（✗）及实际值

**JSONPath 示例：**

```
响应体：{"code":0,"data":{"token":"abc","user":{"id":1}}}

提取 token：  $.data.token       → "abc"
提取用户ID：  $.data.user.id     → 1
提取顶层code：$.code             → 0
```

## 环境变量

环境变量用于管理不同部署环境的配置差异，避免在 API 端点中硬编码。

### 配置变量

1. 进入项目「设置」→「环境变量」
2. 选择环境（stage / pre / prod）
3. 添加 Key-Value 对：

| 变量名 | stage 值 | prod 值 |
|--------|----------|---------|
| `base_url` | `https://api-staging.example.com` | `https://api.example.com` |
| `token` | `test-token-123` | `prod-token-456` |

4. 保存

### 在 API 中使用变量

在 URL、Headers、Body 中使用 `{{variable_name}}` 引用变量：

```
URL: {{base_url}}/api/v1/users
Header: Authorization: Bearer {{token}}
Body: {"env": "{{env_name}}"}
```

运行时选择对应环境，变量自动替换为对应值。

## 场景编排（DAG）

场景编排是平台的核心功能，通过**有向无环图（DAG）**将多个 API 调用串联成完整的测试流程。

### 创建场景

1. 左侧导航点击「场景编排」
2. 点击「新建场景」
3. 填写场景名称和描述
4. 进入 DAG 画布编辑器

### 画布操作

| 操作 | 方式 |
|------|------|
| 添加节点 | 从左侧 API 列表拖拽到画布 |
| 移动节点 | 拖拽节点 |
| 连接节点 | 从节点右侧输出端口拖到另一节点左侧输入端口 |
| 删除节点 | 选中节点 → 按 Delete 键 |
| 删除连线 | 点击连线 → 按 Delete 键 |
| 缩放 | 滚轮缩放 / 右下角缩放按钮 |
| 平移 | 空白处拖拽 |
| 全局视图 | 点击右下角「适应视图」按钮 |
| 保存 | 点击右上角「保存」按钮（或 Ctrl/Cmd+S） |

### 节点配置

点击画布上的节点打开配置面板：

- **API 端点选择**：从项目的 API 列表中选择
- **URL 覆盖**：可以覆盖端点默认 URL（支持 `{{变量}}`）
- **Headers 覆盖**：追加或覆盖请求头
- **Body 覆盖**：覆盖请求体（支持 `{{变量}}`）
- **断言配置**：同 API 调试中的断言，用于场景中的检查点

### 节点间数据传递

连接两个节点后，可以在**连线属性**中配置数据映射，将上游节点的响应值传递给下游节点。

**配置步骤：**

1. 点击两节点之间的连线
2. 添加「数据映射」规则：

| 字段 | 说明 | 示例 |
|------|------|------|
| 来源路径（Source Path）| 上游节点响应的 JSONPath | `$.data.token` |
| 目标变量（Target Key）| 下游节点使用的变量名 | `auth_token` |

3. 下游节点中使用 `{{auth_token}}` 引用此变量

**典型场景示例：**

```
[登录接口] → 提取 $.data.token → auth_token → [获取用户信息] → {{auth_token}}
```

即：先调用登录接口获取 Token，再将 Token 自动注入到后续所有需要认证的接口。

### 运行场景

1. 点击右上角「运行」按钮
2. 选择运行环境（stage / pre / prod）
3. 点击确认
4. 查看运行结果：
   - 每个节点显示执行状态（成功 ✓ / 失败 ✗ / 跳过）
   - 点击节点展开详细结果（请求详情、响应、断言结果）
   - 整体耗时与成功/失败统计

**执行顺序**：按照 DAG 拓扑排序执行，确保上游节点完成后才执行下游节点。

### AI 智能生成场景

无需手动拖拽，用自然语言描述测试需求，AI 自动生成场景。

**使用步骤：**

1. 在场景列表页点击「AI 生成」
2. 在输入框中描述测试场景，例如：

   ```
   测试用户注册流程：先注册一个新用户，然后用注册的账号登录，
   最后验证能否正确获取用户信息。
   ```

3. 点击生成，等待 AI 返回
4. 在画布中预览生成的场景
5. 可手动调整节点和连线
6. 保存场景

> **提示：** 描述越详细，生成效果越好。建议包含：要测试的功能、涉及的接口、数据流向和预期验证点。

## RAG 知识库

RAG（检索增强生成）知识库用于让 AI 理解您的业务文档，提升 AI 场景生成的准确性。

### 入库文档

1. 进入「知识库」页面
2. 点击「上传文档」
3. 支持输入文本内容（API 文档、接口说明、业务规则等）
4. 点击「入库」，系统自动分块并生成向量

### 查询知识库

1. 在「知识库」搜索框输入问题
2. 系统检索相关文档片段，并结合 LLM 生成答案

> **建议入库内容：**
> - API 接口说明文档
> - 业务流程说明
> - 数据字典
> - 测试用例规范

## LLM 配置与用量统计

### 配置 LLM

> 仅超级管理员可修改全局 LLM 配置。

1. 左侧导航点击「LLM 管理」
2. 在「配置」标签页设置：

| 字段 | 说明 |
|------|------|
| Provider | 模型提供商（如 openai） |
| 模型名称 | 如 `gpt-4o-mini`、`gpt-4o` |
| API Key | OpenAI 或兼容 API 的密钥 |
| API Base URL | 可选，用于 OpenRouter / Azure / 本地代理 |
| Temperature | 生成温度（0~2，越低越确定） |
| Max Tokens | 单次最大 Token 数 |

3. 保存配置（立即生效，无需重启）

### 用量统计

切换到「用量统计」标签查看：

- **时间维度图表**：过去 N 天每日 Token 消耗趋势
- **调用来源统计**：各功能模块（AI 生成、RAG 等）的消耗占比
- **明细记录**：每次调用的模型、Token 数、来源接口、时间

## 常见问题

### Q：登录后页面一直跳转到登录页？

Token 可能已过期（默认 24 小时）。重新登录即可。如需延长有效期，请联系管理员修改 `ACCESS_TOKEN_EXPIRE_MINUTES` 配置。

### Q：AI 场景生成结果不准确？

1. 确保 LLM 已正确配置并且 API Key 有效
2. 尝试在知识库中上传更多业务文档
3. 在描述中提供更多上下文，例如指定使用哪些接口

### Q：场景运行时提示变量未找到？

检查：
1. 环境变量是否已在对应环境（stage/pre/prod）中配置
2. 变量名拼写是否与 `{{变量名}}` 中一致（大小写敏感）
3. 节点间数据映射的「目标变量名」是否与使用的 `{{变量名}}` 一致

### Q：导入 cURL 后 URL 中的域名是否会被自动替换为变量？

不会自动替换，需要手动编辑端点，将固定域名替换为 `{{base_url}}` 等变量形式。

### Q：RAG 查询没有返回期望的内容？

1. 确认相关文档已成功入库（检查 `/rag/health` 状态）
2. 尝试调整查询语句，使其更接近入库文档的语言风格
3. 如果文档很长，可以尝试减小 `CHUNK_SIZE` 或调整 `CHUNK_OVERLAP` 后重新入库

### Q：运行场景时某个节点报 timeout 或连接错误？

1. 检查对应环境的 `base_url` 变量是否正确指向可访问的服务地址
2. 确认服务在目标环境中正常运行
3. 检查网络连通性（后端发起请求，需确保后端服务器能访问目标 API）
